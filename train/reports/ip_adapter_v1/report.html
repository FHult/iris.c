Notes

Inference uses the training-matching forward pass: _flux_forward_no_ip collects Q vectors and h_final from a clean Flux run; all 25 IP contributions are summed and added to h_final before norm_out/proj_out. This exactly matches the training computation.
_flux_forward_with_ip (block-by-block injection) is not used — it was never part of the training graph and produces degenerate output with the current weights. Fixing this requires retraining: see TRAIN-6 in BACKLOG.md.
The adapter time (7.1s) is lower than baseline (10.2s) because MLX compiles Metal kernels on the first (baseline) pass; the adapter pass benefits from the warm kernel cache.
